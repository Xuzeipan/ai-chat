## 2. 模式配置说明

### 普通聊天模式
- **适用场景**：日常对话、一般性问题
- **特点**：友好、简洁、通用
- **上下文长度**：10 条消息

### 前端导师模式
- **适用场景**：前端技术问题、代码学习
- **特点**：专业、实用、有代码示例、引导思考
- **上下文长度**：15 条消息（需要更多上下文理解问题）

### 代码审查模式
- **适用场景**：代码质量检查、性能优化
- **特点**：严格、批判性、给出改进建议
- **上下文长度**：20 条消息（需要完整的代码上下文）

## 3. 模式设计原则

### System Prompt 设计要点
1. **明确角色**：告诉模型"你是谁"
2. **设定行为**：规定模型如何回答
3. **控制范围**：限制模型的回答边界
4. **具体要求**：给出具体的回答格式和要求

### Context Length 选择
- **短对话**（5-10）：简单问答、快速响应
- **中等对话**（10-15）：技术讨论、需要上下文
- **长对话**（15-20）：代码审查、复杂问题

## 5. 模式配置文件结构

建议将模式配置放在单独的文件中：

```
src/
├── config/
│   └── modes.ts          # 模式配置文件
├── types/
│   └── index.ts          # 类型定义
└── App.tsx               # 主应用
```
